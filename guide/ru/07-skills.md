# Скиллы

Скиллы Claude Code — это короткие инструкции, которые меняют поведение бота в задачах определённого типа. Не путать с MCP: MCP — это руки (программы-инструменты), скилл — это правила в голове.

## Что такое скилл

Скилл — это файл на markdown с инструкцией. Например, скилл `avoid-ai-writing` говорит: «когда правишь текст, убирай AI-штампы вроде „давайте рассмотрим", „стоит отметить", „не только X, но и Y"».

Когда бот берётся за задачу подходящего типа, он подгружает нужный скилл и работает по его правилам. Сам пользователь скилл не вызывает руками — бот определяет по контексту, какие скиллы применить.

Скилл — это не запрос к API и не запуск программы. Это просто текст, который бот читает перед задачей и держит в голове на время её выполнения.

## Чем отличается от MCP

| | MCP | Скилл |
|---|---|---|
| Что это | Программа-инструмент | Текстовая инструкция |
| Когда срабатывает | Когда бот вызывает функцию | Перед задачей определённого типа |
| Что делает | Реальное действие в мире (запрос в API, рисует картинку) | Меняет поведение бота |
| Пример | `pollinations-image` — рисует картинку | `avoid-ai-writing` — правит текст |
| Где живёт | Отдельный процесс на сервере | Markdown-файл в `~/.claude/skills/` |

Простыми словами: MCP даёт боту новую способность что-то сделать. Скилл говорит боту, как именно это делать, когда уже умеешь.

## Полезные встроенные скиллы

Что есть в наличии у Claude Code:

- **`avoid-ai-writing`** — чистит текст от AI-штампов, делает речь живее. Пригодится, когда просишь написать пост, статью или письмо.
- **`simplify`** — упрощает код, выкидывает лишние абстракции, сокращает.
- **`init`** — генерирует `CLAUDE.md` для нового проекта (это файл с инструкцией для самого Claude — как работать с этим конкретным кодом).
- **`review`** — проверяет твой pull request: ищет баги, плохой стиль, типичные ошибки.
- **`security-review`** — ищет уязвимости в коде. SQL-инъекции, утечки секретов в логи, небезопасное хранение паролей.

Каждый скилл — это маленький файл с правилами. Можно посмотреть, какие правила применяются, открыв сам файл.

## Как установить

На сегодня скиллы устанавливает владелец вручную. Складываются в общую папку `~/.claude/skills/` на сервере, и оттуда их видят все пользователи бота.

В будущем добавим возможность гостям подключать скиллы себе — каждый сможет иметь свой набор правил, не пересекающийся с другими.

Если нужен какой-то конкретный скилл — попроси владельца. Скорее всего, либо такой уже есть, либо допишем за полчаса.

## Можно ли написать свой скилл

Да. Это просто markdown-файл с инструкцией, никакого программирования.

Структура простая:

1. **Название скилла** — короткое, в kebab-case (с дефисами), например `pricing-page-style`
2. **Когда срабатывает** — описание задач, на которых скилл должен подключаться
3. **Что делает** — конкретные правила, которые бот должен соблюдать
4. **Примеры «как надо» и «как не надо»** — чем точнее, тем лучше

Бот может написать тебе скилл по описанию задачи:

> **Ты:** Напиши скилл, который требует, чтобы все цены в моих текстах оформлялись одинаково: «1 200 руб./мес», а не «1200 р/м» или «$15/month». Скилл должен срабатывать, когда я правлю или пишу тексты про тарифы.
>
> **Бот:** Готово. Скилл `pricing-page-style.md`, положи в `~/.claude/skills/`. Содержание:
>
> [текст скилла]

Дальше показываешь владельцу — он положит в общую папку, и бот начнёт его применять.

## Пример скилла

Реалистичный мини-скилл `pricing-page-style.md`. Так выглядит обычный файл в `~/.claude/skills/`:

```markdown
# pricing-page-style

Срабатывает, когда правлю или пишу тексты про тарифы, цены, прайсы.

## Правила

- Сумма всегда с пробелом-разделителем тысяч: `1 200`, не `1200`.
- Валюта — слово «руб.», не символ. Доллары и евро — символом ($, €).
- Период через слэш и сокращение: `руб./мес`, `руб./год`, `$/мес`.
- Никаких «р/м», «руб в месяц», «месячно».

## Как надо

- 1 200 руб./мес
- 990 руб./год
- $15/мес

## Как не надо

- 1200р/м
- 990 рублей в год
- 15 USD per month

## Если непонятно

Спроси пользователя, какой формат он хочет, прежде чем переписывать
весь прайс.
```

Десять строк правил — и бот перестаёт допускать разнобой в оформлении цен.
