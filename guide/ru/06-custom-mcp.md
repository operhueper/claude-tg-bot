# Свои MCP

Когда готовых MCP не хватает — пишется свой. Это не страшно: маленький файл с парой функций. Ниже — зачем, как и с чего начать.

## Зачем

Готовые MCP закрывают популярные сервисы: Gmail и Notion через [Composio](./05-composio.md), генерация картинок через `pollinations-image`, исполнение команд в твоём контейнере. Этого хватает в большинстве случаев.

Свой MCP пригодится, когда:

- у тебя внутренний API компании (свой бекенд, корпоративная CRM)
- сервис, для которого нет ни прямого MCP, ни поддержки в Composio
- хочешь дать боту свою специфическую функцию (например, ходить в твою базу клиентов и считать статистику особым способом)
- нужен инструмент, который пока никто не написал

Шаблоны рабочих MCP лежат прямо в репозитории бота — можно копировать и править под себя.

## Простейший MCP

MCP — это маленькая программа. Чаще всего на TypeScript или Python. Она запускается как отдельный процесс. Бот общается с ней через стандартный ввод-вывод (stdin/stdout) — пишет туда запросы в формате JSON, читает оттуда ответы.

У MCP есть **инструменты** — функции, которые бот может вызывать. Каждый инструмент описывается схемой:

- название (например, `get_weather`)
- что принимает на вход (например, город — строка)
- что возвращает (например, температура — число, описание — строка)

Когда бот видит запрос пользователя «какая погода в Москве», он смотрит список доступных инструментов, находит подходящий `get_weather`, вызывает его с аргументом `city: "Москва"` и получает ответ.

## Минимальный пример

Скелет MCP на TypeScript с одним инструментом `hello`. Полностью рабочей программой не является — это структура, по которой можно писать дальше.

```typescript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

// Создаём сервер: имя и версия
const server = new Server(
  { name: "my-custom-mcp", version: "0.1.0" },
  { capabilities: { tools: {} } }
);

// Регистрируем список инструментов
server.setRequestHandler("tools/list", async () => ({
  tools: [
    {
      name: "hello",
      description: "Здоровается по имени",
      inputSchema: {
        type: "object",
        properties: {
          name: { type: "string", description: "Имя человека" }
        },
        required: ["name"]
      }
    }
  ]
}));

// Что делать, когда бот вызвал инструмент
server.setRequestHandler("tools/call", async (request) => {
  if (request.params.name === "hello") {
    const userName = request.params.arguments.name;
    return {
      content: [{ type: "text", text: `Привет, ${userName}!` }]
    };
  }
  throw new Error("Неизвестный инструмент");
});

// Запускаем
const transport = new StdioServerTransport();
await server.connect(transport);
```

40 строк, один инструмент. Дальше добавляешь свои функции по той же схеме.

## Как добавить в бота

После того как MCP написан и протестирован:

1. Положить папку с кодом в репозиторий бота — например, `my_custom_mcp/` рядом с готовыми `pollinations_mcp/` и `send_file_mcp/`.
2. Открыть `mcp-config.ts` (это локальный файл на сервере, в гит он не коммитится). Добавить туда блок с описанием — путь к файлу, как запускать.
3. Прописать тулзы MCP в `/root/.claude/settings.json` под `permissions.allow` — иначе бот скажет «нет прав».
4. Перезапустить бот:

```bash
systemctl restart claude-tg-bot
```

После этого бот видит новые инструменты и сам начинает их вызывать, когда видит подходящие запросы пользователей.

## Шаги когда хочешь свой MCP

Чек-лист, чтобы ничего не упустить:

1. **Сформулируй**, что должен уметь MCP. Какие инструменты, что принимают, что возвращают. Чем точнее опишешь — тем меньше будет переделок.

2. **Напиши боту:**

   > Помоги написать MCP для нашей внутренней CRM. Нужно два инструмента: получить список клиентов с фильтром по городу, добавить заметку к клиенту по id. API живёт на `https://crm.mycompany.ru`, ключ в переменной `CRM_API_KEY`.

   Бот напишет код, положит в твой контейнер, протестит на простых случаях.

3. **Проверь у себя.** Попроси бота вызвать новые инструменты и посмотри, что приходит в ответ.

4. **Покажи владельцу.** Если интеграция полезная не только тебе — подключим в общий бот для всех пользователей. Если только тебе — оставим в твоём контейнере.

## Куда смотреть

В репозитории `claude-tg-bot` уже лежат четыре готовых MCP — компактные, можно использовать как пример:

- `ask_user_mcp/` — задаёт пользователю вопрос с кнопками. Самый маленький, хорош как первый пример.
- `send_file_mcp/` — отправляет файл в чат. Видно, как MCP кладёт «запрос» на диск, а бот его читает.
- `pollinations_mcp/` — генерация картинок через внешнее API. Пример работы с HTTP-запросами и сохранением результата.
- `openrouter_image_mcp/` — то же, но для платного OpenRouter. Пример работы с API-ключами через переменные окружения.

Открой любой и пройдись по `index.ts` — там всё на месте: список инструментов, обработчики, запуск сервера.
