## Сайт-визитка

Сайт https://jinru.pro/ksyusha раздаётся **прямо из этой папки** (через `/opt/ksyusha-cv/server.js`, порт 8765, проксируется nginx на jinru.pro).

- `ksyusha-vizitka.html` — главная страница (отдаётся на `/ksyusha/`).
- Любой другой файл из этой папки доступен по `/ksyusha/<имя_файла>` — картинки, css, js и т.д.
- В html ссылайся на ассеты относительными путями (`src="ksyusha-photo.jpg"`, без слеша в начале).
- `CLAUDE.md` (этот файл) скрыт от внешнего мира — наружу не отдаётся.

**Деплой не нужен.** Любая правка файла в этой папке сразу live на сайте — никаких scp, ssh, systemctl restart. Просто редактируй файлы здесь и обновляй страницу.

Если бот говорит «нужно скопировать на сервер» — это галлюцинация: ты УЖЕ на сервере, файлы здесь и есть на сервере, и сайт читает их отсюда же.


## Память

Профиль автоматически инжектируется в системный промпт при старте каждой сессии.
Файл профиля: `memory/893951298/profile.md` (в этой папке).

При просьбе «запомни что я ...» сначала пойми тип факта:

- **Стабильное** (имя, любимый цвет, постоянные предпочтения) — Edit profile.md.
- **Эфемерное** (болею сегодня, занята делом X на этой неделе, в отпуске) — НЕ редактируй profile.md, оно само попадёт в graph.json с датой и со временем устареет.

Если сомневаешься — спроси «это надолго или на сегодня?».

Граф памяти и саммари сессий хранятся в `memory/893951298/` (graph.json, sessions/, transcripts/).

## Граф памяти — активное использование

Файлы: `memory/893951298/graph.json`, `memory/893951298/sessions/`, `memory/893951298/topics-index.md`.
Профиль: `memory/893951298/profile.md` (инжектируется в системный промпт автоматически).

**Когда и как читать:**
- При старте содержательного разговора: читай `topics-index.md` → найди тему → читай последние 1-2 сессии
- Поиск по теме: `grep -i "слово" memory/893951298/topics-index.md` → читай файлы из списка
- Если тема не найдена в индексе: `grep -rl "слово" memory/893951298/sessions/`
- Если Ксюша упоминает что-то без контекста (название, имя, место) → читай `graph.json`
- Если Ксюша говорит «мы обсуждали» / «в прошлый раз» / «помнишь» → читай последние сессии по теме
- **Верификация:** если Ксюша говорит что-то, что противоречит ноде в графе — уточни явно («у меня записано X, это всё ещё актуально?») и обнови ноду
- Ноды типа `runbook_step`, `deploy_quirk`, `infra` — технический мусор. Старше 30 дней → можно удалять при очистке

**Классификация фактов:**
- Стабильное (имя, стек, постоянные предпочтения) → `profile.md`
- Эфемерное (болею, в отпуске, занята на неделе) → только сессия, не трогай профиль
- Стабильный факт проекта → `graph.json`
- Инсайт / рефлексия → `mind/reflections/YYYY-MM-DD-название.md`
- «Запомни», «важно», «мысль», «инсайт» → всегда сохраняй в файл, не только в граф
- Лучше прочитать и убедиться, чем переспрашивать то, что уже обсуждали

## Канал @otklicker — генерация постов

Когда тебя просят сгенерировать или предложить пост в канал @otklicker:

1. Проверь, что уже опубликовано: WebFetch `https://t.me/s/otklicker`. Сверься с `otklicker/posted.md` (он может отставать).
2. Открой `otklicker/prewritten_queue.json` — там 7 заготовок. Если из них что-то ещё не выходило, **предложи пользователю взять следующий по порядку**.
3. Если очередь исчерпана или пользователь просит свежий пост — генерируй новый по правилам:
   - голос и длина: `otklicker/channel-voice.md`
   - бренд-токены (если речь про визуал/обложку): `otklicker/brand-tokens.md`

4. **Никогда не публикуй без явного «ОК» от пользователя.** Сначала покажи текст, дождись подтверждения.
5. Публикация: бот @proboiAI_bot — админ канала с can_post_messages. После «ОК» отправь через `sendMessage` Bot API в chat_id `-1003964848299`.
6. После публикации — допиши строку в `otklicker/posted.md` с датой/заголовком.
