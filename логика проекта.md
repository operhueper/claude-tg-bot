## Три уровня доступа

### Уровень 1 — Owner (Евгений)

- `settingSources: ["user", "project"]` — читает `~/.claude/CLAUDE.md` (глобальные инструкции)
- `allowedPaths`: широкие (/opt, /root, /home, /tmp...)
- Команды: все включая `/restart`, `/reloadbot`
- **Системный промпт** (`OWNER_SAFETY_PROMPT`): английский, базовые правила безопасного удаления файлов и ограничения путей
- **CLAUDE.md** (`workspace/CLAUDE.md`): очень богатый — FitCoach, Отклыкер, инфра-маршрутизация, граф памяти, маркетинговые книги, параллельные агенты, карта admin-операций

### Уровень 2 — Legacy Guest (Ксения)

- `settingSources: ["project"]` — не читает `~/.claude` владельца
- `allowedPaths`: только workspace-ksenia
- Команды: нет `/reloadbot`, есть `/restart`
- **Системный промпт** (`KSENIA_GUEST_SAFETY_PROMPT`): русский, детальный — само-расширение (pip/apt/bun без вопросов), список инструментов, изоляция, правила удаления
- **CLAUDE.md** (`workspace-ksenia/CLAUDE.md`): богатый — FitCoach с базой упражнений, Откликер с брендбуком, Second Brain, граф памяти, сайт jinru.pro/ksyusha
- **Модель: claude-sonnet-4-6** (та же что у владельца — самая дорогая)

### Уровень 3 — New Guest (Артём + 5 неизвестных)

- `settingSources: ["project"]`
- `allowedPaths`: только /opt/vault/{userId} + /tmp/telegram-bot
- Команды: нет `/restart`, нет `/reloadbot`
- **Системный промпт** (`NEW_GUEST_SAFETY_PROMPT`): самый богатый по функциям — скиллы (`ls skills/` при каждом запросе), адаптация стиля под пользователя, сессии с темами, публичная страница `https://jinru.pro/u/{userId}/`, антипетля, форматы файлов
- **CLAUDE.md**: генерируется из шаблона (`src/templates/guest-claude-md.ts`) — стандартный
- **Модель: deepseek-chat** (не Claude; vision → gemini-2.5-flash)

## Как строится контекст каждого запроса

```
systemPrompt (из config.ts, зависит от профиля)
  + profile.md (если существует в vault/memory/{userId}/)
  + buildMemoryContext() → топ-20 нод графа + активные цели (≤2000 символов)

[Current date/time: ...] + сообщение пользователя
```

Память инжектируется только в начале новой сессии. Формат:
```markdown
## Что я знаю о тебе

### Активные цели
- [daily] ...

### Недавние достижения
- ...

### Ключевые факты
- 👤 Евгений — role: developer
- 📁 claude-tg-bot — status: active
```

## Sandbox-контейнеры (добавлено 2026-05-06)

### Модель изоляции

Каждый гостевой пользователь получает Docker-контейнер как среду исполнения кода. Claude при этом остаётся общим (HTTP-вызовы в OpenRouter/Anthropic) — контейнер не держит Claude-процесс.

```
Сообщение тестера
  → Бот (один процесс) → OpenRouter/DeepSeek (HTTP)
  ← DeepSeek: tool_call Bash("ls /workspace")
  → docker exec claude-user-{userId} bash -c "ls /workspace"  ← выполняется в контейнере
  ← результат → DeepSeek → ответ пользователю
```

Контейнер в покое: **~8-10 MB** (sleep infinity + kernel overhead). Тяжёлый процесс появляется только когда пользователь запускает что-то сам (сайт, скрипт).

### Образ контейнера: `claude-user-sandbox:latest`

Base: debian:bookworm-slim. Установлено: Bun, Node.js 20, Python 3, git, nginx, build-essential, sqlite3, базовые cli-инструменты. Образ собирается на сервере, 314 MB на диске.

### Монтирование внутри контейнера

| Хост | Контейнер | Права |
|---|---|---|
| Docker volume `claude-user-{userId}-data` | `/workspace` | RW — persistent хранилище пользователя |
| `/root/.claude/` | `/root/.claude/` | RO — credentials + settings для Claude CLI |
| `/var/lib/claude-bot/dropbox/{userId}/` | `/tmp/dropbox` | RW — file-drop для ask-user/send-file MCP |

### Лимиты

| Профиль | CPU | RAM | docker.sock |
|---|---|---|---|
| Owner (Евгений) | без лимита | без лимита | ✅ примонтирован + /var/log:ro |
| Guest | 1.0 ядро | 512 MB + 512 MB swap | ❌ |

### Жизненный цикл контейнера

`absent` → (первое сообщение) → `running` → (15 мин без сообщений) → `paused` → (24ч) → `stopped`

- `paused`: cgroup-freezer, RAM не освобождается, resume мгновенный
- `stopped`: RAM освобождается, workspace-volume сохраняется
- `remove`: только явным действием владельца

### Invite-only система

Новый пользователь пишет боту → бот отвечает «доступ закрыт» и сохраняет `PendingInvite` в `/var/lib/claude-bot/pending/{userId}.json` → Евгений получает Telegram-уведомление с кнопками **Одобрить / Отклонить** → при одобрении: пользователь добавляется в `system/users.json` как `new_guest` с `containerEnabled: true`, контейнер создаётся автоматически при первом сообщении.

### Ёмкость текущего сервера (1 vCPU, 1.9 GB RAM)

- Зарегистрировано: **20-30 тестеров** (большинство молчат)
- Одновременно активны (пишут боту): **5-7**
- Одновременно гоняют тяжёлый код (npm install, серверы): **2-3**
- Узкое место: **CPU (1 ядро)**, не RAM

При апгрейде до CX22 (4 vCPU, 8 GB, €7.49/мес): 50-100 активных тестеров.
