## Два уровня доступа

### Owner (Евгений)

- `settingSources: ["user", "project"]` — читает `~/.claude/CLAUDE.md`
- `allowedPaths`: широкие (/opt, /root, /home, /tmp...)
- Команды: все, включая `/restart`, `/reloadbot`
- Системный промпт (`buildOwnerSafetyPrompt`): правила безопасного удаления, ограничения путей, блок ANTI-HALLUCINATION ON ERRORS
- CLAUDE.md (`workspace/CLAUDE.md`): богатый контекст — FitCoach, Отклыкер, инфра, граф памяти, параллельные агенты

### Guest (все остальные)

- `settingSources: ["project"]` — `~/.claude` владельца не читается
- `allowedPaths`: только `/opt/vault/{userId}/` + `/tmp/telegram-bot`
- Команды: нет `/restart`, нет `/reloadbot`
- Системный промпт (`buildNewGuestSafetyPrompt`): скиллы, адаптация стиля, сессии, веб-страница, изоляция, само-расширение, антипетля, блок «Анти-галлюцинация ошибок»
- CLAUDE.md: генерируется из шаблона `src/templates/guest-claude-md.ts` (содержит раздел «Анти-галлюцинация ошибок»)
- Модель: deepseek-chat через `api.deepseek.com/anthropic`

**Исключение — Ксения (`893951298`):** Guest-профиль, но модель `claude-sonnet-4-6` (явное решение). `allowedPaths` расширены — включают `CLAUDE_WORKING_DIR` (owner workspace). `disallowedTools` не применяется (Ксения на Sonnet — WebSearch работает).

> Полные тексты промптов — в `src/config.ts` (`buildOwnerSafetyPrompt`, `buildNewGuestSafetyPrompt`, `buildGroupSystemPrompt`).

## Sandbox-контейнеры

Каждый гость получает Docker-контейнер как среду выполнения кода. Claude остаётся общим (HTTP-вызовы) — контейнер не держит Claude-процесс.

```
Сообщение гостя
  → Бот → DeepSeek/Anthropic (HTTP)
  ← DeepSeek: tool_call Bash("ls /workspace")
  → docker exec claude-user-{userId} bash -c "ls /workspace"
  ← результат → ответ пользователю
```

**Образ:** `claude-user-sandbox:latest` (debian:bookworm-slim, Bun, Node 20, Python 3, git, nginx, sqlite3). ~314 MB.

**Лимиты гостя:** 1.0 ядро, 512 MB RAM + 512 MB swap. Owner — без лимитов, docker.sock примонтирован.

**Жизненный цикл:** `absent` → `running` → (15 мин без сообщений) → `paused` → (24ч) → `stopped`. Workspace-volume сохраняется.

**Invite-only:** новый пользователь → бот сохраняет `PendingInvite` → Евгений получает уведомление с кнопками «Одобрить / Отклонить» → при одобрении пользователь добавляется в `users.json`, контейнер создаётся при первом сообщении. При одобрении ставится `onboardingComplete: false` — новый гость проходит 6-шаговый онбординг, после чего флаг переключается в `true`.

**Файлы:** `Dockerfile.user`, `src/containers/manager.ts`, `src/containers/spec.ts`, `src/containers/paths.ts`, `src/containers/invites.ts`, `src/engines/openrouter.ts` (Bash tool → docker exec).

## Тулзы у гостей и owner

DeepSeek-гости (все кроме Ксении) имеют в профиле `disallowedTools: ["WebSearch"]`. Это поле проставляется в `getUserProfile()` (`src/config.ts`) и пробрасывается в опции SDK `query()` через `src/session.ts`.

**Причина:** `api.deepseek.com/anthropic` не проксирует серверные tools Anthropic (WebSearch, WebFetch). При попытке Claude Code дёрнуть WebSearch через этот эндпоинт DeepSeek возвращает `does not support this tool_choice`. Это вызывало галлюцинацию бота — он объяснял ошибку придуманными причинами («российский IP», «нет прав»).

**Альтернатива для гостей:** WebFetch + Bash curl на `html.duckduckgo.com` — работает через контейнер напрямую, без серверных tools Anthropic.

**Owner и Ксения** (Anthropic API напрямую) — без ограничений, WebSearch работает.

## Метеринг и дашборд

Токены каждого пользователя пишутся в `metering.sqlite` через `src/metering.ts`. Перехват в `src/session.ts` (Anthropic/DeepSeek) и `src/engines/openrouter.ts`. Источники: `bot-anthropic`, `bot-deepseek`, `bot-openrouter`.

Встроенный HTTP-сервер на порту 3848 (`src/dashboard-server.ts`). Маршруты: `/` (лендинг), `/dashboard` (Mini App), `/api/me`, `/api/admin/all`. Аутентификация через Telegram `initData` (HMAC-SHA256). URL команды `/dashboard` пока ведёт на `ksenyaenbom.ru/dashboard` — обновить при переезде на `proboi.site`.
