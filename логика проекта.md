# Логика проекта claude-tg-bot

*Обновлено: 2026-05-07*

---

## 1. Что это такое

Telegram-бот, который даёт людям доступ к AI-ассистенту (Claude или DeepSeek) прямо через телефон. Пишешь в Telegram — бот передаёт запрос в AI, тот выполняет команды, ищет в интернете, работает с файлами и отвечает обратно. Бот работает на TypeScript с Bun и живёт на двух серверах: боевой (jinru) и тестовый (proboi-bot).

## 3. Как работает запрос

```
Сообщение в Telegram
  ↓
Проверка доступа (allowlist)
  ↓
Загрузка профиля пользователя (getUserProfile)
  ↓
Сборка системного промпта:
  - buildOwnerSafetyPrompt / buildNewGuestSafetyPrompt
  - + profile.md (если есть в vault/memory/{userId}/)
  - + buildMemoryContext() → топ-20 фактов из graph.json + активные цели (≤2000 символов)
  ↓
AI-запрос (DeepSeek или Claude через Agent SDK)
  ↓
Стриминг ответа → Telegram
  ↓
Запись токенов в metering.sqlite
```

Память вставляется только в начале новой сессии. При команде `/new` сессия сбрасывается.

## 4. Что может делать гость

**Есть:**
- Отправлять текст, голос, фото, документы, видео
- Bash-команды на сервере в своей папке `/opt/vault/{userId}/`
- Установка пакетов без вопросов: `pip install`, `npm install`, `bun add`, `apt-get install`
- WebFetch — читать любую страницу по URL
- Поиск через `curl` + DuckDuckGo (вместо WebSearch)
- Создание и редактирование файлов в своей папке
- Публичная веб-страница: `https://proboi.site/u/{userId}/`
- Долгосрочная память: profile.md, graph.json, sessions/
- Персональные скиллы: файлы в `skills/`, которые гость сам обучает
- Команды: `/start`, `/new`, `/stop`, `/status`, `/resume`, `/retry`, `/dashboard`, `/goals`

**Нет:**
- WebSearch — DeepSeek не поддерживает серверные инструменты Anthropic, вызов упадёт с ошибкой
- Доступ к чужим vault-папкам
- Доступ к коду бота (`/opt/claude-tg-bot/src/`, `.env`, `mcp-config.ts`)
- Команды `/restart` и `/reloadbot`
- Чтение `~/.claude` владельца (`settingSources: ["project"]` — изоляция памяти)

**Лимиты:**
- **maxTurns: 20** за один запрос — защита от бесконечных поисковых петель
- **Лимит поиска: 3 попытки** — если за 3 поиска ничего не нашёл, честно говорит об этом и останавливается

## 5. Что может делать owner

- Все возможности гостей, плюс:
- WebSearch работает без ограничений
- `settingSources: ["user", "project"]` — читает `~/.claude/CLAUDE.md` (личная память, скиллы)
- Широкие пути: `/opt`, `/root`, `/home`, `/tmp`, `/var/tmp`, `/usr/local`, `/etc`
- Прямое редактирование `.env`, `mcp-config.ts`, `src/**`, `system/users.json`, systemd-конфигов
- Команды: `/restart`, `/reloadbot` (дополнительно к общим)
- Нет лимита ходов (maxTurns не установлен)
- Нет rate limit (или настраивается через `.env`)

## 7. Как устроена память

Каждый пользователь имеет изолированную папку памяти: `/opt/vault/{userId}/memory/{userId}/`.

- **profile.md** — стабильные факты: имя, стиль общения, предпочтения. Читается в начале содержательной сессии.
- **graph.json** — граф фактов и связей (имена, проекты, предпочтения). Используется для контекста.
- **sessions/** — история разговоров по темам. Файлы формата `YYYY-MM-DD-HHmm-тема.md`.
- **topics-index.md** — индекс тем для быстрого поиска нужной сессии.
- **skills/** — пользователь обучает ассистента своим инструкциям. Каждый скилл — отдельный `.md`-файл.

В начале новой сессии бот собирает `buildMemoryContext()`: берёт топ-20 нод графа и активные цели (не больше 2000 символов) и вставляет в системный промпт.

Сессии хранятся в `/tmp/claude-telegram-session-{userId}.json`. Каждый пользователь — отдельный файл, изоляция полная.

## 8. Как работает Bash у гостей

**Важно: Docker не используется для DeepSeek-гостей.**

Bash-команды гостей выполняются напрямую на сервере в их рабочей папке `/opt/vault/{userId}/`. Никакого контейнера между гостем и сервером нет.

Ограничения задаются через:
- `allowedPaths` в профиле — только своя vault-папка и `/tmp/telegram-bot`
- `checkCommandSafety()` — проверка опасных команд (rm и т.п.) против списка разрешённых путей
- Системный промпт — явный запрет читать чужие vault'ы и код бота

В коде есть `ContainerManager` (`src/containers/manager.ts`) — он написан для будущего использования и задействован только в `src/engines/openrouter.ts` для vision-запросов. Для DeepSeek-гостей контейнеры сейчас не используются. На тестовом сервере образ `claude-user-sandbox:latest` вообще не собран.

## 9. Серверы и деплой

| | Prod (jinru) | Test (proboi-bot) |
|---|---|---|
| Хост | `root@5.223.82.96` | `root@89.167.125.175` |
| Бот | `@proboiAI_bot` | `@ORCH7_bot` |
| Путь | `/opt/claude-tg-bot/` | `/opt/claude-tg-bot/` |
| Сервис | `systemctl restart claude-tg-bot` | то же |
| Логи | `/var/log/claude-tg-bot.log` | то же |

**Деплой:**
```bash
# На prod
rsync -az --exclude node_modules --exclude .git --exclude .env ./ root@5.223.82.96:/opt/claude-tg-bot/
ssh root@5.223.82.96 'cd /opt/claude-tg-bot && bun install && systemctl restart claude-tg-bot'

# На test
rsync -az --exclude node_modules --exclude .git --exclude .env ./ root@89.167.125.175:/opt/claude-tg-bot/
ssh root@89.167.125.175 'cd /opt/claude-tg-bot && bun install && systemctl restart claude-tg-bot'
```

**Никогда не rsync `.env`** — у каждого сервера свой токен бота. Синхронизация `.env` перезапишет токен и вызовет 409 Conflict на prod.

**После `bun install` на сервере** — нужно восстановить glibc-бинарник Claude SDK (musl/glibc ловушка):
```bash
cp /root/.local/share/claude/versions/2.1.126 \
  /opt/claude-tg-bot/node_modules/@anthropic-ai/claude-agent-sdk-linux-x64-musl/claude
chmod +x /opt/claude-tg-bot/node_modules/@anthropic-ai/claude-agent-sdk-linux-x64-musl/claude
systemctl restart claude-tg-bot
```
