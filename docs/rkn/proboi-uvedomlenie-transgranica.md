## 7. Категории передаваемых персональных данных

При трансграничной передаче передаётся **исключительно содержимое конкретного запроса** Пользователя:

| Категория | Состав |
|---|---|
| Содержимое сообщения | Текст сообщения, направленного Пользователем в адрес AI-ассистента |
| Аудиоконтент | Голосовое сообщение Пользователя в формате OGG/MP3 (для целей транскрипции) |
| Графический контент | Изображение, направленное Пользователем (для целей анализа) |
| OAuth-метаданные (Composio) | Идентификатор Telegram-пользователя в форме `tg_<id>`, технические токены OAuth для конкретного провайдера (Google), запрашиваемые scopes |

**НЕ передаётся при трансграничной передаче:**

- Telegram user_id, username, имя профиля (за исключением случая Composio, где идентификатор используется для разделения OAuth-сессий);
- история переписки за пределами текущего запроса;
- содержимое персонального хранилища (vault), за исключением файлов, которые Пользователь явно прикрепил к запросу;
- платёжные данные;
- IP-адрес;
- иные данные о профиле и активности Пользователя.

## 8. Сведения о наличии в иностранных государствах, не обеспечивающих адекватной защиты прав субъектов ПДн, мер защиты передаваемых персональных данных

### 8.1. Меры, принимаемые на стороне Оператора

1. **Минимизация состава передаваемых данных.** AI-провайдерам передаётся только содержимое конкретного запроса; идентификационные данные Пользователя (Telegram user_id, имя) не передаются.
2. **Изоляция запросов.** Запросы Пользователей передаются обезличенно; провайдер не имеет возможности связать содержание запроса с конкретным Пользователем по данным, передаваемым Оператором.
3. **Договорные основания.** Оператор использует только тех AI-провайдеров, чьи публичные оферты (Terms of Service / Data Processing Addendum) содержат обязательства по обеспечению конфиденциальности и безопасности обрабатываемых данных, а также запрет на использование клиентских данных для обучения моделей по умолчанию (для платных API).
4. **Шифрование при передаче.** Все запросы передаются по протоколу HTTPS (TLS 1.2 и выше).
5. **Контроль за хранением.** Оператор использует только API-эндпоинты провайдеров; стационарное хранение пользовательских данных на серверах провайдеров не предусмотрено (за исключением краткосрочного логирования на стороне провайдера, ограниченного его собственной политикой).
